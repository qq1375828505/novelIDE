# 网文写作IDE — Windows桌面版技术方案（单机独立版）
## 基于 Electron + Monaco Editor 从零搭建

---

## 1. 产品定位
一款面向网络文学作者的专业桌面端写作IDE，**完全单机运行**，不依赖任何云端服务（除用户自行配置的AI API外）。作品数据、AI配置、设置全部保存在本地，Windows版和Android版互为独立软件，无账号互通、无自动同步。

**Windows版定位为主力创作端**：负责长篇写作、作品管理、章节整理、大纲设定、AI辅助、精修审阅、导出投稿。

产品核心原则：
- **写小说优先**：正文编辑、章节管理、自动保存、历史快照优先级最高。
- **AI是助手，不是主界面**：AI能力围绕续写、润色、精修、标题、大纲、爽点检查服务。
- **本地安全优先**：正文、设定、历史版本、API配置均保存在本机。
- **分阶段开发**：V1先完成写作核心闭环，V2增强网文能力，V3再做高级IDE能力。

## 2. 技术栈
| 层级 | 技术 | 说明 |
|------|------|------|
| 框架 | Electron 28+ | 独立桌面应用 |
| 前端 | TypeScript + React 18 + TailwindCSS | 完全自定义UI |
| 编辑器 | Monaco Editor | 网文化定制 |
| AI/搜索/审查 | 直接调用用户配置的API | 本地模型(Ollama/LM Studio) + 云端API(DeepSeek/Claude等) |
| 数据存储 | SQLite索引 + Markdown正文文件 + JSON/YAML配置 | SQLite管理索引和配置，正文以Markdown文件保存，便于备份和导出 |
| 构建工具 | Vite + esbuild | |

## 3. 核心架构

### 3.1 完全单机
```
┌─────────────────────────────────────────┐
│           Windows单机版                  │
│  ┌─────────────┐  ┌─────────────────┐   │
│  │  Electron   │  │ SQLite+MD+JSON │   │
│  │  UI层       │  │  本地数据存储   │   │
│  └──────┬──────┘  └─────────────────┘   │
│         │                                │
│  ┌──────┴──────────────────────────┐    │
│  │      业务逻辑层（主进程）         │    │
│  │  - 章节管理                      │    │
│  │  - AI调用（直接请求用户API）      │    │
│  │  - 联网搜索（直接请求搜索API）    │    │
│  │  - 事实提取/冲突检测（本地规则+可选本地模型）│
│  │  - 钩子追踪                      │    │
│  │  - 一键精修（直接请求用户API）    │    │
│  │  - Agent运行（本地解释器）        │    │
│  │  - 模型路由（本地配置）           │    │
│  │  - 费用统计（本地计算）           │    │
│  └──────────────────────────────────┘    │
│                                          │
│  外部连接：用户自行配置的AI API、搜索API   │
│  无：云端同步、账号系统、后端服务          │
└─────────────────────────────────────────┘
```

### 3.2 与Android版关系
**完全独立**：
- 无账号体系，两端不互通
- 无数据同步，作品各自保存在各自设备
- AI配置各自独立设置（各自存各自的API Key）
- 如需作品迁移，通过导出/导入文件（Markdown/JSON）手动传输

### 3.3 工作台布局
Windows版采用“三栏 + 底部面板”布局，保证正文编辑区域始终是主角。

```
┌──────────────┬──────────────────────────────┬──────────────────┐
│ 左侧导航      │          中间正文编辑器        │ 右侧写作参考       │
│ 作品/卷/章    │          Monaco Editor        │ 角色/设定/大纲     │
│ 搜索/统计     │                              │ 伏笔/备注          │
├──────────────┴──────────────────────────────┴──────────────────┤
│ 底部面板：AI对话 / 精修 / 审查 / 模型配置 / 日志                  │
└─────────────────────────────────────────────────────────────────┘
```

| 区域 | 功能 | 说明 |
|------|------|------|
| 左侧导航 | 作品、卷、章节树、全文检索 | 管理作品结构 |
| 中间编辑器 | 正文写作 | 保持最大空间，专注写作 |
| 右侧参考栏 | 角色卡、设定卡、大纲、伏笔、备注 | 写作时随手查看，不打断正文 |
| 底部面板 | AI对话、精修、审查、模型、日志 | 工具型功能，不抢正文区域 |

**底部面板Tab：**
- `对话`：AI助手聊天
- `精修`：选中文本精修、章节精修
- `审查`：设定提醒、爽点检查、水文检测
- `模型`：ModelHub配置
- `日志`：保存记录、AI调用记录、错误信息
- `Agent`：后续阶段开放

### 4.2 设定提醒展示
V1阶段不做重型“全文剧情冲突检测”，先做更稳定的**设定提醒**。

**展示方式：列表展示 + 点击定位**（不实时下划线）：
- 作者手动建立角色设定、修为设定、地点设定、伏笔设定
- 系统通过关键词和简单规则发现可能冲突
- 底部面板或右侧参考栏显示提醒列表
- 点击提醒项 → 编辑器滚动到对应位置并高亮

示例：
```text
已有设定：陈默 = 筑基三层
正文出现：陈默释放出金丹威压
提醒：可能冲突，陈默当前设定为“筑基三层”。
```

## 5. 功能模块

### 5.0 Windows版开发阶段

| 阶段 | 目标 | 功能范围 |
|------|------|----------|
| V1 | 先能用 | 写作核心闭环：作品、卷、章节、正文编辑、自动保存、AI续写/润色、导出 |
| V2 | 更好用 | 角色卡、设定卡、设定提醒、番茄爽点、水文检测、标题生成、写作统计 |
| V3 | 更专业 | AgentForge、全文语义审查、本地轻量模型、插件系统、高级模型路由 |

**V1必须优先完成：**
1. 新建作品
2. 新建卷
3. 新建章节
4. 章节树
5. Monaco正文编辑器
6. 自动保存
7. 本地历史快照
8. AI对话
9. 选中文字润色
10. AI续写
11. 导出Markdown
12. API Key配置
13. 番茄基础预设


### 5.1 章节管理
- 卷→章→节三级树形结构
- 拖拽排序、合并、拆分
- SQLite记录章节索引和顺序
- 正文保存为Markdown文件
- 自动保存 + 本地历史快照（非Git）

### 5.2 角色卡系统（V2）
- 角色信息卡片
- 关系图谱（D3.js）
- 出场统计

### 5.3 联网搜索 (NovelSearch)（V2）
- 选中文字→右键"联网查证"
- 多搜索源（百科、图片、学术）
- 结果保存到本地"参考资料"库
- **直接请求搜索API**，无中间后端

### 5.4 自定义大模型 (ModelHub)
- 本地模型：Ollama、LM Studio（直接连接localhost）
- 云端API：DeepSeek、Claude、GPT等（用户自行配置Key）
- 智能路由规则（本地配置）
- 费用统计（本地计算，基于配置的费率）
- **API Key加密存储**：Windows Credential Manager

### 5.5 设定提醒与剧情连贯性（分阶段）

**第一阶段 V1：设定提醒**
- 作者手动标记关键设定（如"陈默修为：筑基三层"）
- 字符串匹配与规则提醒（文中出现"金丹威压"时提示可能冲突）
- 手动添加钩子，系统追踪闲置章节数
- 不承诺自动理解全文，只做稳定的提醒工具

**第二阶段 V2：轻量本地模型辅助**
- 集成BERT-tiny等轻量模型做实体识别
- 自动提取人物、地点、修为等实体
- 人工确认后入库

**第三阶段 V3：全文语义冲突检测（实验室功能）**
- 完整的语义理解
- 自动钩子识别

### 5.6 自定义Agent (AgentForge)（V3）
- Agent市场（本地YAML预设 + 可导入社区配置）
- Agent配置编辑器（System Prompt、记忆策略、触发条件）
- 工作流编排
- **Agent运行在本地**，直接调用用户配置的模型API

### 5.7 一键精修 (AutoPolishEngine)（V1简版，V2增强）
- V1：选中文字 → 精修 → 生成修改版 → 替换/插入/复制/取消
- V1快捷预设：保持原意、增强爽感、减少废话、加强画面感、优化对话
- V2：语义分块对比审阅（按段落/对话块分块，非按行）
- V2：左右分栏展示、差异高亮、逐块采纳/跳过
- V2：8维度精修（语病、节奏、文风、冗余、对话、描写、钩子、战力）
- **直接调用用户配置的模型API**

### 5.8 写作统计（V2）
- 字数、时段、词频、角色出场
- 本地计算，无上传


### 5.9 自动保存与历史快照（V1核心）
小说软件必须优先保护正文安全，自动保存优先级高于所有高级AI功能。

**保存规则：**
- 输入停止800ms后自动保存当前章节
- 每30秒强制保存一次当前章节
- 切换章节前自动保存
- 关闭软件前自动保存
- AI替换正文前生成快照
- 每章默认保留最近50个历史快照

**历史版本：**
```text
第128章 终焉之樱
├── 当前版本
├── 10分钟前
├── 30分钟前
├── 昨天 22:18
└── 前天 19:03
```

**恢复方式：**
- 预览历史版本
- 恢复为当前版本
- 复制历史片段到当前章节

## 6. 番茄小说专属模块 (TomatoStudio) — 新增

### 6.1 定位
针对**番茄小说平台**快节奏、强爽点、高密度的写作风格，提供一键切换的AI预设和专属Agent。番茄读者偏好：黄金三章强钩子、每章3-4个爽点、节奏快不拖沓、标题党吸睛。

4. 玄幻无敌·横推诸天

6. 年代重生·整顿亲情流
7. 恶毒女配·全家反派洗白流
8. 无CP大女主·搞钱流

**V1番茄核心按钮：**
- 生成黄金三章开局
- 检查本章爽点
- 生成爆款标题

## 9. 隐私设计

### 9.1 三级隐私模式
- **严格**：所有AI调用强制走本地模型（Ollama/LM Studio），不上传任何内容
- **均衡**：AI调用前自动脱敏（人名→代号），脱敏后上传云端
- **云端**：直接上传（速度快）

### 9.2 脱敏规则（本地执行）
```typescript
// 上传云端前，本地自动脱敏
function sanitize(text: string, novelId: string): string {
  // 人名→[角色A]、地名→[地点1]、作品名→[作品标题]
  // 返回后本地还原
}
```

### 9.3 API Key安全
- Windows Credential Manager 加密存储
- 内存中使用时解密，不落盘明文

## 13. 小说记忆系统（NovelMemory）— 从Android版采纳

### 13.1 设计理念

为每部小说维护一个**持久化的创作记忆文件**，记录小说的完整状态，让任何AI工具都能读懂当前创作进度。

### 13.2 记忆文件结构

```
小说记忆文件 (Novel Memory File)
├── 1. 作品信息（书名、字数、章节数、主线大纲）
├── 2. 卷章结构（分卷列表、每章状态和字数）
├── 3. 最近章节摘要（最近5章）
├── 4. 角色状态（角色卡完整信息、属性标签）
├── 5. 设定状态（世界观、金手指、战力体系）
├── 6. 地点信息
├── 7. 势力信息
├── 8. 重要道具
├── 9. 伏笔追踪（未回收/已回收、闲置超过10章自动警告⚠️）
└── 10. 参考资料
```

### 13.3 自动更新机制

- **触发时机**：每次编辑器保存章节后自动调用 `NovelMemory.autoUpdate()`
- **存储路径**：`Documents/NovelIDE/{novelId}/memories/{novelId}_memory.txt`
- **缓存机制**：内存缓存，避免短时间内重复生成

### 13.4 AI上下文注入

AI对话时自动将记忆文件注入 system prompt：
```
systemPrompt = "你是网文写作助手。\n\n小说记忆文件：\n{memory_content}"
```

所有AI调用点都已集成：
- 底部面板AI对话
- 选中文字润色/续写
- 爽点检查/水文检测/标题生成

### 13.5 导出包含

每次导出ZIP中自动包含 `小说记忆文件.txt`，方便：
- 其他AI工具读取完整上下文
- 跨设备迁移时恢复创作状态
